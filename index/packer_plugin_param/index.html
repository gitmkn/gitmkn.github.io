Packer插件字段说明

2023年11月24日 · arunma.cn

翻译来源于AI

必输字段

参数名 类型 描述
secret_id string 腾讯云的 Secret ID。您可以直接设置它，或设置 TENCENTCLOUD_SECRET_ID 环境变量。
secret_key string 腾讯云的 Secret Key。您可以直接设置它，或设置 TENCENTCLOUD_SECRET_KEY 环境变量。
region string 您要启动 CVM 的地区。您应参考区域和可用区参数。
zone string 您要启动 CVM 的可用区。您应参考区域和可用区参数。
instance_type string 您要启动 CVM 的实例类型。您应参考实例类型参数。
image_name string 您要创建的自定义镜像名称，它应由不超过60个字符的字母、数字或减号组成。

可选字段

参数名 类型 描述
cvm_endpoint string 连接云端点，如果是 TCE 云，则应设置为 TCE CVM 端点。
vpc_endpoint string 连接云端点，如果是 TCE 云，则应设置为 TCE VPC 端点。
security_token string STS 访问令牌，可以通过模板设置或通过导出环境变量（例如，export SECURITY_TOKEN=value）。
assume_role TencentCloudAccessRole 提供此块时，Packer 将尝试使用提供的凭据进行假定角色。
role_arn string 要假定的角色的 ARN。可以从 TENCENTCLOUD_ASSUME_ROLE_ARN 获取。
session_name string 进行 AssumeRole 调用时要使用的会话名称。可以从 TENCENTCLOUD_ASSUME_ROLE_SESSION_NAME 获取。
session_duration int 进行 AssumeRole 调用时的会话持续时间（以秒为单位）。其值范围为 0 到 43200（秒），默认值为 7200 秒。可以从 TENCENTCLOUD_ASSUME_ROLE_SESSION_DURATION 获取。
profile string 凭据配置文件中的配置文件名称。也可以从 TENCENTCLOUD_PROFILE 环境变量获取。如果未设置，则默认使用 tccli configure 创建的默认配置文件。如果未设置，则默认为 default。
shared_credentials_dir string 共享凭据的目录。也可以从 TENCENTCLOUD_SHARED_CREDENTIALS_DIR 环境变量获取。如果未设置，则默认为 ~/.tccli。
image_description string 镜像描述。长度不超过 60 个字符。
force_poweroff bool 在创建镜像时是否执行强制关机。默认值为 false。
sysprep bool 在创建 Windows 镜像时是否启用 Sysprep。
image_copy_regions []string 在创建镜像后将复制到的区域。
image_share_accounts []string 在创建镜像后将共享到的账户。
image_tags map[string]string 将应用于结果镜像的键/值对标签。
associate_public_ip_address bool 是否为您的 CVM 分配公共 IP。默认值为 false。
source_image_id string 要从中创建自定义镜像的基础镜像 ID。
source_image_name string 要从中创建自定义镜像的基础镜像名称。冲突时使用 SourceImageId。
instance_charge_type string CVM 的计费类型，值可以是 POSTPAID_BY_HOUR（默认）或 SPOTPAID。
instance_name string 实例名称。
disk_type string CVM 将启动的根磁盘类型。默认为 CLOUD_PREMIUM。您可以参考磁盘类型。
disk_size int64 CVM 将启动的根磁盘大小。值范围：
data_disks []tencentCloudDataDisk 在创建镜像之前，将一个或多个数据磁盘添加到实例。注意，如果源镜像具有数据磁盘快照，则此参数将被忽略，实例将使用源镜像中的数据磁盘设置，在这种情况下，disk_type 参数将用于所有数据磁盘，每个数据磁盘大小将使用源镜像中的原始值。数据磁盘允许以下参数：
vpc_id string 指定 CVM 将启动的 VPC。
vpc_name string 指定将创建的 VPC 名称。如果 vpc_id 未设置，Packer 将为您创建一个名为此参数的 VPC。
subnet_id string 指定 CVM 将启动的子网。
subnet_name string 指定子网名称。如果 subnet_id 未设置，Packer 将为您创建一个名为此参数的子网。
cidr_block string 指定 vpc 的 cider 块。
subnect_cidr_block string 指定子网的 cider 块。
internet_charge_type string CVM 的 Internet 费用类型，值可以是 TRAFFIC_POSTPAID_BY_HOUR、BANDWIDTH_POSTPAID_BY_HOUR 或 BANDWIDTH_PACKAGE。
internet_max_bandwidth_out int64 CVM 将启动的最大出站带宽（以 MB 为单位）。值可以设置在 1 到 100 之间。
bandwidth_package_id string 当 internet_charge_type 为 BANDWIDTH_PACKAGE 时，需要指定 bandwidth_package_id。
security_group_id string 指定 CVM 将启动的安全组。
security_group_name string 指定安全组名称。如果 security_group_id 未设置，Packer 将为您创建一个名为此参数的安全组。
user_data string 用户数据。
user_data_file string 用户数据文件。
host_name string 主机名。
cam_role_name string CAM 角色名称。
run_tags map[string]string 在启动实例时附加到实例的标签。这些标签不会应用于结果镜像。
run_tag []{key string, value string} 与 run_tags 类似，但允许您以单独可重复的块形式定义键和值。在 HCL2 模式下，动态块允许您以编程方式创建这些

连接配置参数

参数名 类型 描述
communicator string Packer目前支持三种通信器：none - 不使用通信器，ssh - 建立SSH连接，通常是默认的选项，winrm - 建立WinRM连接。此外，一些构建器具有自定义通信器，如Docker构建器的 “docker” 通信器使用 docker exec 和 docker cp 执行脚本和复制文件。
pause_before_connecting string ex: “1h5m2s”，我们建议在客户机的引导脚本中的最后一步启用SSH或WinRM，但有时可能存在等待的竞争条件，需要Packer在尝试连接到客户机之前等待一段时间。使用模板选项 pause_before_connecting 可以解决这种情况。默认情况下，没有暂停。例如，如果将 pause_before_connecting 设置为 10m，Packer将正常检查是否可以连接。但一旦连接尝试成功，它将断开连接，然后等待10分钟，然后再连接到客户机并开始配置。
ssh_host string SSH 连接地址。通常由构建器自动配置。
ssh_port int SSH 连接端口。默认为 22。
ssh_username string SSH 连接用户名。使用 SSH 时必须提供。
ssh_password string 用于身份验证的明文密码。
ssh_ciphers []string 覆盖 Golang 默认支持的加密算法。默认值为 [“aes128-gcm@openssh.com”, “chacha20-poly1305@openssh.com”, “aes128-ctr”, “aes192-ctr”, “aes256-ctr”]。
ssh_clear_authorized_keys bool 如果为 true，则 Packer 将尝试从 ~/.ssh/authorized_keys 和 /root/.ssh/authorized_keys 中删除其临时密钥。这是一个大多数情况下仅起到美化作用的选项，因为不管设置为 true 还是 false，Packer 都会从主机系统中删除临时私钥（除非用户设置了 -debug 标志）。默认值为 false；目前仅适用于安装了 sed 的客户机。
ssh_key_exchange_algorithms []string 如果设置，则 Packer 将覆盖 Golang 默认支持的密钥交换（kex）算法。可接受的值包括：“curve25519-sha256@libssh.org”、“ecdh-sha2-nistp256”、“ecdh-sha2-nistp384”、“ecdh-sha2-nistp521”、“diffie-hellman-group14-sha1” 和 “diffie-hellman-group1-sha1”。
ssh_certificate_file string 用于身份验证的用户证书文件路径。路径中可以使用 ~ 表示当前用户的主目录。
ssh_pty bool 如果为 true，则请求一个 PTY 进行 SSH 连接。默认为 false。
ssh_timeout string ex:“1h5m2s”， 等待 SSH 可用的时间。Packer 使用此选项来确定主机何时已启动，因此通常会很长。例如：10m。默认为 5m，除非设置了 ssh_handshake_attempts。
ssh_disable_agent_forwarding bool 如果为 true，则禁用 SSH 代理转发。默认为 false。
ssh_handshake_attempts int SSH 连接成功后尝试握手的次数。默认为 10，除非设置了 ssh_timeout。
ssh_bastion_host string 用于实际 SSH 连接的跳板主机。
ssh_bastion_port int 跳板主机的端口。默认为 22。
ssh_bastion_agent_auth bool 如果为 true，则使用本地 SSH 代理进行跳板主机身份验证。默认为 false。
ssh_bastion_username string 连接跳板主机的用户名。
ssh_bastion_password string 用于身份验证的跳板主机密码。
ssh_bastion_interactive bool 如果为 true，则使用键盘交互进行跳板主机身份验证。
ssh_bastion_private_key_file string 用于身份验证的 PEM 编码私钥文件路径。路径中可以使用 ~ 表示当前用户的主目录。
ssh_bastion_certificate_file string 用于身份验证的用户证书文件路径。路径中可以使用 ~ 表示当前用户的主目录。
ssh_file_transfer_method string 文件传输方式，可选值
ssh_keypair_name string 如果指定，则这是用于 SSH 连接的密钥。密钥必须与远程加载的密钥对名称匹配。默认情况下，这是空白的，除非使用 ssh_password。当使用 ssh_keypair_name 时，必须指定 ssh_private_key_file 或 ssh_agent_auth。
ssh_private_key_file string 用于身份验证的 PEM 编码私钥文件路径。路径中可以使用 ~ 表示当前用户的主目录。
ssh_agent_auth bool 如果为 true，则使用本地 SSH 代理进行连接身份验证。不会创建临时密钥对，并且 ssh_password 和 ssh_private_key_file 的值将被忽略。必须设置环境变量 SSH_AUTH_SOCK 才能正常使用此选项。
